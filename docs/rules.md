## 1. 游戏概述

### 1.1 基本规则
- **玩家人数**：2-10人（标准满桌9人）
- **用牌**：标准52张扑克牌，无大小王
- **目标**：用2张底牌 + 5张公共牌中的任意组合，组成最好的5张牌
- **获胜方式**：
  1. 摊牌(Showdown)时牌型最大
  2. 通过下注迫使所有对手弃牌

### 1.2 牌型组成方式
每位玩家有 **2张底牌(Hole Cards)**，桌面有 **5张公共牌(Community Cards)**。
玩家从这7张牌中选出 **最佳5张** 组合。可以：
- 2张底牌 + 3张公共牌
- 1张底牌 + 4张公共牌
- 0张底牌 + 5张公共牌（"打桌面"）

## 2. 牌型排名系统

从高到低共10种牌型：

### 2.1 皇家同花顺 (Royal Flush)
- **牌型**：A-K-Q-J-10，全部同花色
- **示例**：A♠ K♠ Q♠ J♠ 10♠
- **概率**：0.000154% (1/649,740)
- **备注**：最高牌型，无法被击败。同一手牌中两个皇家同花顺平分底池

### 2.2 同花顺 (Straight Flush)
- **牌型**：5张连续且同花色的牌
- **示例**：9♥ 8♥ 7♥ 6♥ 5♥
- **概率**：0.00139% (1/72,193)
- **比较**：最高牌点大者胜
- **A的特殊性**：A可以作为A-K-Q-J-10的最高端，也可作为5-4-3-2-A的最低端

### 2.3 四条 (Four of a Kind)
- **牌型**：4张相同点数的牌
- **示例**：K♠ K♥ K♦ K♣ 3♠
- **概率**：0.024% (1/4,165)
- **比较**：四条点数大者胜；若相同（公共牌四条），则第5张牌(kicker)大者胜

### 2.4 满堂红/葫芦 (Full House)
- **牌型**：3条 + 1对
- **示例**：Q♠ Q♥ Q♦ 7♣ 7♠
- **概率**：0.144% (1/694)
- **比较**：三条部分点数大者胜；三条相同时比对子

### 2.5 同花 (Flush)
- **牌型**：5张同花色的牌（不连续）
- **示例**：A♣ J♣ 8♣ 4♣ 2♣
- **概率**：0.197% (1/509)
- **比较**：从最高牌点开始逐张比较，直到分出胜负
- **注意**：两个同花时，花色本身没有大小之分

### 2.6 顺子 (Straight)
- **牌型**：5张连续的牌（花色不同）
- **示例**：10♦ 9♠ 8♥ 7♣ 6♦
- **概率**：0.392% (1/255)
- **比较**：最高牌点大者胜
- **A的特殊性**：A-2-3-4-5（"轮子"，最小顺子）和10-J-Q-K-A（"百老汇"，最大顺子）

### 2.7 三条 (Three of a Kind)
- **牌型**：3张相同点数的牌
- **示例**：8♠ 8♥ 8♦ K♣ 4♠
- **概率**：2.11% (1/47)
- **比较**：三条点数大者胜；相同时比较剩余2张踢脚牌

### 2.8 两对 (Two Pair)
- **牌型**：两组对子
- **示例**：A♠ A♥ 5♣ 5♦ 9♠
- **概率**：4.75% (1/21)
- **比较**：先比较高对，再比低对，最后比踢脚牌

### 2.9 一对 (One Pair)
- **牌型**：1组对子
- **示例**：J♠ J♥ A♦ 7♣ 3♠
- **概率**：42.26% (1/2.4)
- **比较**：先比对子点数，再依次比较踢脚牌

### 2.10 高牌 (High Card)
- **牌型**：无任何组合，比最大单牌
- **示例**：A♠ Q♥ 9♦ 6♣ 3♠ ("A高")
- **概率**：50.12% (1/2)
- **比较**：从最高牌点开始逐张比较

### 2.11 牌型速查表
```
皇家同花顺  >  同花顺  >  四条  >  葫芦  >  同花  >  顺子  >  三条  >  两对  >  一对  >  高牌
```

## 3. 游戏流程

### 3.1 完整一手牌的流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Preflop │ -> │   Flop   │ -> │   Turn   │ -> │  River   │ -> │ Showdown │
│  翻牌前  │    │  翻牌    │    │   转牌   │    │   河牌   │    │  摊牌    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
  2张底牌         3张公共牌        1张公共牌        1张公共牌        比大小
  第1轮下注       第2轮下注        第3轮下注        第4轮下注        决定胜负
```

### 3.2 详细步骤

**阶段0：发牌前**
- 庄家按钮(BTN)确定
- 小盲注(SB)和大盲注(BB)强制下注
- 每位玩家发2张底牌（仅自己可见）

**阶段1：Preflop（翻牌前）**
- 从UTG(枪口位)开始，顺时针行动
- 公共牌：0张
- 可用信息：仅自己的2张底牌
- 主要决策：根据起手牌强度、位置决定入池

**阶段2：Flop（翻牌）**
- 发3张公共牌，牌面朝上
- 从SB开始（庄家按钮后第一位存活玩家）
- 公共牌：3张
- 可用信息：2+3=5张牌
- 主要决策：评估牌力、听牌、对手范围

**阶段3：Turn（转牌）**
- 发第4张公共牌
- 下注限额翻倍（限注游戏）
- 公共牌：4张
- 可用信息：2+4=6张牌
- 主要决策：重新评估牌力，此时强牌变得更强

**阶段4：River（河牌）**
- 发第5张（最后一张）公共牌
- 公共牌：5张
- 可用信息：7张牌选5张，牌力已确定
- 主要决策：价值下注、诈唬、抓诈

**阶段5：Showdown（摊牌）**
- 最后下注轮结束后，存活玩家摊牌
- 最后主动下注者先亮牌
- 牌型最大者赢得底池
- 牌型相同则平分底池（Chop）

## 4. 位置系统

### 4.1 9人桌标准位置

```
┌─────────────────────────────────────────────┐
│                 UTG (枪口)                    │
│             UTG+1 (枪口+1)                    │
│         MP (中间位 / Middle Position)          │
│        HJ (劫位 / Hijack)                     │
│       CO (关煞位 / Cutoff)                    │
│      BTN (庄位 / Button) ─── 最佳位置          │
│     SB (小盲 / Small Blind)                   │
│    BB (大盲 / Big Blind)                      │
└─────────────────────────────────────────────┘

Dealer (D) ← 庄家按钮，每手牌顺时针移动一位
```

### 4.2 位置的价值排名（从好到差）

| 位置 | 缩写 | 特点 | 相对优势 |
|------|------|------|----------|
| 庄位 | BTN | 翻牌后总是最后行动 | 最强位置 |
| 关煞位 | CO | 仅次于庄位 | 可以偷盲 |
| 劫位 | HJ | 中间偏后 | 可玩中等牌 |
| 中位 | MP | 中等位置 | 需较紧范围 |
| 枪口+1 | UTG+1 | 早期位置 | 需紧范围 |
| 枪口 | UTG | 最差非盲注位置 | 需最紧范围 |
| 大盲 | BB | 已投入1个大盲 | 可防守 |
| 小盲 | SB | 最差位置+已投入 | 最难盈利 |

### 4.3 短桌位置（6-max）

```
UTG → MP → CO → BTN → SB → BB
```

6人桌每个位置都更接近庄位，范围可以放宽。

### 4.4 位置对策略的影响

**有位置(In Position)**：翻牌后最后行动
- 可以看到对手的行动后再做决策
- 可以控制底池大小
- 更容易实施诈唬
- 范围可以更宽

**无位置(Out of Position)**：翻牌后优先行动
- 信息劣势
- 容易被施加压力
- 更难实现胜率
- 需要更紧的范围

## 5. 下注系统

### 5.1 游戏类型

#### 无限注德州扑克 (No-Limit Texas Hold'em) — 最主流
- 随时可以全下(All-in)
- 最小加注：等于上一注的加注幅度
- 最大加注：所有筹码
- 主流赛事和现金局的标准玩法

#### 底池限注 (Pot-Limit)
- 最大加注 = 当前底池大小
- 常用于PLO，德州较少

#### 固定限注 (Fixed-Limit)
- 每轮下注和加注有固定限制
- Preflop/Flop：1个小注
- Turn/River：1个大注（通常2x小注）

### 5.2 下注动作

| 动作 | 英文 | 说明 |
|------|------|------|
| 弃牌 | Fold | 放弃当前手牌和已投入筹码 |
| 过牌 | Check | 不下注，让给下一位（前提：无人下注） |
| 跟注 | Call | 匹配当前最大下注 |
| 下注 | Bet | 第一个下注（翻牌后） |
| 加注 | Raise | 在他人下注基础上提高注额 |
| 再加注 | Re-raise / 3-Bet | 在加注基础上再次加注 |
| 全下 | All-in | 投入所有剩余筹码 |

### 5.3 加注尺度标准

```
最小加注 = 当前注额 + (当前注额 - 上一注额)
        = 当前注额 + 加注幅度

示例：盲注 1/2
- 第一位加注到 6（加注幅度 = 4）
- 最小再加注 = 6 + 4 = 10
- 而非 6 + 6 = 12（虽然通常下注更大）
```

### 5.4 标准翻牌前加注尺度

| 情景 | 尺度 | 说明 |
|------|------|------|
| 标准开池 | 2.5-3x BB | 常规开池加注 |
| 有人平跟后 | 3-4x + 1x/每个平跟者 | 惩罚平跟 |
| 3-Bet | 3x 原加注 (IP) / 4x (OOP) | 再加注 |
| 4-Bet | 2.2-2.5x 3-Bet | 4-Bet尺度较小 |
| 全下 | 20BB以下可考虑 | 短码推 |

### 5.5 翻牌后下注尺度

| 情景 | 尺度（底池百分比） | 用途 |
|------|---------------------|------|
| 小额下注 | 25-33% | 干燥牌面持续下注 |
| 标准下注 | 50-66% | 常规价值/诈唬 |
| 大额下注 | 75-100% | 极化范围 |
| 超额下注 | 125-200% | 极限极化（坚果或空气） |

## 6. 底池与边池

### 6.1 底池分类

**主底池(Main Pot)**：所有玩家都有资格赢取的筹码

**边池(Side Pot)**：当一名玩家全下且筹码不足跟注完整加注时创建

### 6.2 边池计算示例

```
玩家A：100筹码（全下）
玩家B：200筹码（全下）
玩家C：500筹码

玩家A全下100 → 主底池 = 100×3 = 300（3人都参与）
玩家B加注到200 → 边池1 = 100×2 = 200（B和C争夺）
玩家C跟注200 → 玩家C剩余300

结果：
- 主底池 300：A、B、C都有资格争夺
- 边池1 200：仅B、C有资格争夺
- 玩家C保留 300（未投入）

摊牌顺序：从小到大比
1. A只能赢主底池
2. B能赢主底池和边池
3. C能赢所有底池
```

### 6.3 底池计算公式

```
def calculate_side_pots(players):
    # 按投入金额排序
    all_in_amounts = sorted(set(p.all_in_amount for p in players if p.all_in_amount > 0))
    
    pots = []
    prev_amount = 0
    
    for amount in all_in_amounts:
        pot = 0
        eligible = []
        for p in players:
            contribution = min(p.total_bet, amount) - min(p.total_bet, prev_amount)
            if contribution > 0:
                pot += contribution
            if p.total_bet >= amount and not p.folded:
                eligible.append(p)
        pots.append({'amount': pot, 'eligible': eligible})
        prev_amount = amount
    
    return pots
```

## 8. 赔率与期望值

### 8.1 底池赔率 (Pot Odds)

```
底池赔率 = 跟注金额 / (当前底池 + 跟注金额)

示例：
底池 = 100，对手下注 50
跟注赔率 = 50 / (100+50+50) = 50/200 = 25%

如果胜率 > 25%，长期跟注是+EV
```

### 8.2 隐含赔率 (Implied Odds)

```
隐含赔率考虑未来可能赢取的额外筹码

需要赢取 = (跟注金额) - (胜率 × 当前底池)
额外需要赢取 / 胜率 = 需要在后续街赢取的金额

适用于：听坚果同花、听顺子等隐蔽性强的手牌
不适用于：明显的听牌（对手可能不会支付）
```

### 8.3 期望值公式 (EV)

```
EV = (胜率 × 赢得金额) - (败率 × 损失金额)

EV(Fold) = 0
EV(Call) = (Win% × Pot) - ((1-Win%) × Call)
EV(Bet) = (Fold% × Pot) + (1-Fold%) × [(Win% × (Pot+Bet)) - ((1-Win%) × Bet)]

+EV = 长期盈利
-EV = 长期亏损
```

### 8.4 常见场景EV速查

**翻牌前全下胜率：**
| 对阵 | 胜率 |
|------|------|
| 大对子 vs 小对子 | 80% vs 20% |
| 对子 vs 两高张 | 55% vs 45% |
| 两高张 vs 两低张 | 65% vs 35% |
| AK vs 任意对子 | 45% vs 55% |
| AK同花 vs 任意对子 | 47% vs 53% |
| 一张高张 vs 对子 | 30% vs 70% |

## 9. 策略框架

### 9.1 起手牌范围 (Starting Hand Ranges)

**9人桌 UTG 开池范围（紧凶风格）：**
```
对子：77+, 即 77/88/99/TT/JJ/QQ/KK/AA
同花：ATs+, KJs+, QJs, JTs, T9s, 98s, 87s
不同花：AQo+

总共约 14-15% 的手牌
```

**BTN 开池范围（可放宽到）：**
```
对子：22+
同花：A2s+, K5s+, Q8s+, J8s+, T7s+, 97s+, 86s+, 75s+, 65s, 54s
不同花：A2o+, K9o+, Q9o+, J9o+, T9o

总共约 40-45% 的手牌
```

### 9.2 不同位置推荐VPIP范围

| 位置 | VPIP (入池率) | PFR (加注率) |
|------|---------------|---------------|
| UTG | 12-15% | 12-15% |
| MP | 16-20% | 15-19% |
| HJ | 20-25% | 18-23% |
| CO | 25-32% | 22-28% |
| BTN | 35-45% | 28-38% |
| SB | 30-40% | 15-25% |
| BB | 25-45% (防守) | 8-12% |

### 9.3 翻牌后决策树

```
翻牌后首次行动：
├─ 击中强牌 (两对+) → 价值下注 (60-80% pot)
├─ 击中中等牌 (顶对好踢脚) → 价值下注 (50-66% pot)
├─ 击中弱牌 (中对/底对) → 过牌控池/小注
├─ 强听牌 (12+ outs) → 下注/加注半诈唬
├─ 弱听牌 (4-8 outs) → 过牌跟注/过牌弃牌
└─ 无击中 → 根据牌面决定是否持续下注(C-bet)

持续下注(C-bet)建议：
├─ 干燥牌面(K72彩虹) → 小注(33%)高频
├─ 湿润牌面(JT9两同花) → 大注(66-75%)低频
└─ 中等奖牌面 → 混合策略
```

### 9.4 GTO vs 剥削策略

**GTO (Game Theory Optimal 博弈论最优)**：
- 不可被剥削
- 不考虑对手弱点
- 适合面对未知/高水平对手
- 使用混合策略（一定频率做不同行动）

**剥削策略 (Exploitative)**：
- 针对对手具体漏洞
- 对手弃牌过多 → 增加诈唬
- 对手跟注过多 → 减少诈唬，加厚价值下注
- 适合已知对手弱点时使用

### 9.5 翻牌分类

| 牌面类型 | 特点 | 策略含义 |
|----------|------|----------|
| 干燥(Dry) | 不连续，无同花听牌 | 高频率小注C-bet |
| 湿润(Wet) | 连张，有同花听牌 | 低频大注，需要有牌 |
| 静态(Static) | 坚果不易变化 | 可慢打 |
| 动态(Dynamic) | 后续牌容易改变牌力 | 保护手牌 |

## 10. 术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 底牌 | Hole Cards | 玩家私有的2张牌 |

## 附录A：牌型判断算法核心逻辑

```
牌型判断步骤：
1. 统计每种点数的数量（pair_count）
2. 统计每种花色的数量（suit_count）
3. 检查是否有同花（任一种花色>=5张）
4. 检查是否有顺子（连续5个点数）
5. 按优先级判断：
   - 同花顺 → 同花 && 顺子
   - 四条   → 任一点数出现4次
   - 葫芦   → 3+2 或 3+3的组合
   - 同花   → 任一种花色>=5
   - 顺子   → 连续点数>=5
   - 三条   → 任一点数出现3次
   - 两对   → 两个点数各出现>=2次
   - 一对   → 任一点数出现>=2次
   - 高牌   → 以上都不满足
```

## 附录B：下注轮状态机

```
[New Hand] → Post Blinds → Deal Hole Cards
    ↓
[Preflop] → Betting Round → 所有人跟注/AI → [Flop]
    否则 → 有人下注无人跟 → 直接赢得底池 → [New Hand]
    ↓
[Flop] → Deal 3 Cards → Betting Round
    ↓
[Turn] → Deal 1 Card → Betting Round
    ↓
[River] → Deal 1 Card → Betting Round
    ↓
[Showdown] → 比较牌型 → 分配底池 → [New Hand]
```
